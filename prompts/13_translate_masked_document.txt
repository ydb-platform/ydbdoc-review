Вы переводите **фрагмент markdown** с маскированными атомами (плейсхолдеры `⟦KIND:n⟧`).

## Плейсхолдеры (критично)

- Каждый токен `⟦KIND:n⟧` (например `⟦LINK:3⟧`, `⟦HTML:12⟧`) **копируйте дословно**.
- **Запрещено:** удалять, переименовывать, менять номер, разбивать, склеивать, заменять на `[…](url)`, `` `…` ``, `<…>` или текст из SOURCE.
- **Число** плейсхолдеров и **порядок** в выводе должны совпадать с входом.
- Если сомневаетесь — оставьте плейсхолдер как в SOURCE и переведите только текст **между** ними.

## Перевод

- Переводите **только** обычный текст между плейсхолдерами на TARGET.
- Сохраняйте markdown-структуру: строки таблиц `| … |`, переносы строк, `##` заголовки, `-` списки.
- **Не** вставляйте markdown-ссылки вместо символа `\` в конце строк shell.
- Не добавляйте `` ``` ``, новые `{% … %}`, префиксы `L00001|`, обёртку «Translation request».
- Не объединяйте и не дробите абзацы и **строки таблиц** без необходимости.

## Что означают маски (не переводите их содержимое)

- `LINK` — markdown-ссылка `[…](url)`
- `HTML` — тег `<…>`
- `CODE` — `` `inline` ``
- `VAR` — `{{ … }}`
- `ANCHOR` — `{#…}`
- `DIPL` — директива `{% … %}`
- `FENCE` — блок `` ``` … ``` ``

## Выход

Верните **только** переведённый markdown-фрагмент (без code fence вокруг всего ответа).
