## Иерархия качества перевода

Каждый фрагмент должен стремиться к уровню **Best**. Используйте критерии ниже.

### Best — можно публиковать без правок

Одновременно выполнены все пункты:

1. **Естественная формулировка.** Текст читается так, будто его писал носитель для технической документации — без кальки с SOURCE.
   - RU: «Для того чтобы удалить файл, необходимо нажать кнопку **Удалить**.»
   - ✓ Best: «To delete a file, click **Delete**.»
   - ✗ Unacceptable: «For the purpose of deleting the file, it is necessary to press the button **Delete**.»

2. **Точный смысл.** Нет добавлений, пропусков и сдвигов акцента.
   - RU: «Операция **необратима**.»
   - ✓ Best: «This operation is **irreversible**.»
   - ✗ Unacceptable: «The operation can be undone.»

3. **Терминология.** Технические термины согласованы с глоссарием (если передан) и единообразны во фрагменте.

4. **Разметка.** Markdown, YFM, якоря `{#...}`, ссылки, fenced-блоки и таблицы сохранены.

5. **Тон.** Тот же регистр и прямота, что в SOURCE — без лишней разговорности и без излишней сухости.

### Satisfactory — допустимо с доработкой

Смысл не искажён, но есть 1–2 мелких недочёта: лёгкая калка, один синоним вместо термина из глоссария, чуть неестественная фраза.

### Unacceptable — недопустимо в выводе

Любой из пунктов делает фрагмент недопустимым:

1. **Смысловая ошибка** — изменён, инвертирован или опущен критичный факт.
2. **Неверный термин** — подставлено другое понятие (role → password, row → string в неверном контексте).
3. **Сломанная разметка** — fence, таблица, директива YFM, якорь или ссылка повреждены; весь ответ обёрнут в `` ``` `` без SOURCE.
4. **Непереведённый текст** вне кода и идентификаторов (кириллица в EN-prose или в EN-комментариях, в том числе «похожие» буквы).
   - ✗ `…r.com #FQDN ВМ host…` (кириллица «ВМ»)
   - ✓ `…r.com #VM FQDN host…`
5. **Неверный CLI-флаг** — пробел внутри имени флага.
   - SOURCE: `bootstrap --uuid <строка>`
   - ✗ EN: `bootstrap -- uuid <string>`
   - ✓ EN: `bootstrap --uuid <string>`
6. **Галлюцинация** — команды, флаги, числа, URL, которых нет в SOURCE.
