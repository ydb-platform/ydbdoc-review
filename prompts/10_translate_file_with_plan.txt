Вы переводите **фрагмент одного markdown-файла** технической документации YDB.

Перед текстом идёт **план по номерам строк** (1-based, как в редакторе): для каждого диапазона указано, что делать.
Следуйте плану **строго**. Не переводите регионы с пометкой «копировать дословно».

## Общие правила

- Выведите **только** переведённый markdown для запрошенного диапазона строк. Без пояснений, без обёртки в ` ``` `.
- Сохраняйте структуру: заголовки, списки, таблицы, пустые строки, Diplodoc-директивы.
- `{% note %}`, `{% cut %}`, `{% list tabs %}`, `{% endlist %}`, `{{ ydb-short-name }}` — синтаксис не менять; переводите только видимый текст внутри.
- Если в тексте встречаются плейсхолдеры вида `<<FENCE_BLOCK_001>>`, **копируйте их дословно**, не меняя ни одного символа. Это маркеры для подстановки исходного кода.
- Блоки `{% list tabs %}…{% endlist %}` в этот запрос **не попадают** (копируются из SOURCE отдельно). Если видите такой блок — скопируйте дословно.
- Якоря `{#...}` в заголовках — **копировать из SOURCE**; переводите только текст заголовка.
- Markdown-ссылки `[текст](url)` — сохранять; переводить текст, URL не ломать.
- Имена вкладок SDK (`- Go`, `- Python`, …), CLI-флаги (`--uuid`), пути, ключи YAML/JSON, идентификаторы типов — **не переводить**.
- Каждый CLI-флаг — один токен: `--uuid`, не `-- uuid`.

## По типам регионов в плане

| План | Действие |
|------|----------|
| prose | Перевести весь диапазон |
| table | Таблица: переводить по ячейкам; ✓ и имена типов (`DyNumber`, …) на тех же местах |
| fence | Блок между ` ``` ` — **дословно**; если в плане перечислены строки с комментариями — перевести **только** текст комментария на этих строках |
| tabs | Весь блок `{% list tabs %}…{% endlist %}` — **дословно** (не должно попадать в запрос; если попал — копировать целиком) |
| diplodoc | Блок note/cut: директивы не трогать, перевести текст внутри |

## Несколько запросов на один файл

Если в user-сообщении указано «запрос K из N» — вы переводите **только** указанный диапазон строк этого файла.
Не добавляйте контент из других частей файла.
