Вы обновляете английскую документацию YDB так, чтобы она отражала **только** изменения в русском файле между merge base и веткой PR.

На входе по порядку:
1) REFERENCE_EN — полная текущая английская статья. Эталон формулировок и **всех** целей ссылок markdown (`](path)` без изменений, если русское изменение явно не требует другого path).
2) RU_DIFF — unified `git diff` русского файла (merge-base..HEAD). Только он задаёт правки в английском.
3) RU_FULL — полный русский файл для контекста.

Правила (строго):
- Один полный английский markdown-документ, без текста до/после.
- Абзацы, строки, заголовки, ячейки таблиц, fence, которые **не** следуют из RU_DIFF, копируйте **дословно** из REFERENCE_EN (пробелы, пустые строки, пунктуация, ссылки).
- **Блоки кода не переводите:** всё между ` ``` ` (bash, sh, yaml, ini, …) — **байт-в-байт из REFERENCE_EN**, если строка не в RU_DIFF; если строка **добавлена/изменена** в RU_DIFF — копируйте команду/конфиг с русского (переводите только `#` комментарии внутри fence).
- Shell/CLI вне fence: на неизменённых строках — **посимвольно из REFERENCE_EN**. На **новых** из RU_DIFF — с русского, с пробелами (`--config-dir /opt/ydb/cfg`). Не подменяйте `--config-dir` на `--yaml-config`.
- **Минимальный scope:** если RU_DIFF затрагивает 1–10 строк, итоговый файл должен почти полностью совпадать с REFERENCE_EN; запрещён полный resync статьи под RU_FULL.
- **Diplodoc tabs:** в разделах, **затронутых** RU_DIFF, сохраняйте все `{% list tabs %}`, `{% endlist %}`, `{% list tabsgroup %}` из RU_FULL (переводите только подписи вкладок). В разделах **вне** RU_DIFF — оставляйте блоки tabs как в REFERENCE_EN (не подтягивайте tabs из RU_FULL).
- **Fences:** сохраняйте ` ``` ` из REFERENCE_EN; для новых блоков — тот же язык fence, что в русском (`yaml`, `bash`, …).
- В **неизменённых** fenced-блоках конфигурации (yaml/ini/toml) **не** меняйте ключи и значения; в новых/изменённых — копируйте структуру из русского, переводите только `#` комментарии при необходимости.
- **Ссылки:** неизменённые — как в REFERENCE_EN; новые с `yandex.cloud/ru/docs/` → `/en/docs/` в английском.
- **Не** перетекайте абзацы, **не** добавляйте/убирайте пустые строки, кроме вставок под строки `+` в RU_DIFF.
- **Не** «улучшайте» английский вне diff.
- **Не** вставляйте русские слова в английский текст.
- **Не** добавляйте пояснения, которых нет в русском diff (например «используя полученный токен», если в RU_DIFF этого нет).

Вывод: только markdown.
